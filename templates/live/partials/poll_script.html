{% comment %} Inline JS que faz polling de cards de jogos em andamento. Incluir uma unica vez no final de paginas que listam matches (``matches/match_list.html``, ``predictions/pool_matches.html``). O script vasculha o DOM em busca de ``
`` com os atributos ``data-match-id`` e ``data-live="1"`` e, a cada 60 segundos, faz um ``fetch`` na rota ``match_card_partial`` correspondente (``/matches//_partial/``), trocando o HTML interno do card pelo retorno. Esta e a unica excecao ao stack "DTL puro, zero JS" do projeto (US-7.4 explicita essa justificativa): placares ao vivo nao podem ser entregues sem algum mecanismo client-side, e a alternativa (server-sent events / websockets) esta fora do escopo. Comportamento: * Apenas cards com ``data-live="1"`` sao polled. Quando o jogo sai de "em_andamento" o backend renderiza o card com ``data-live="0"`` e o script para de atualiza-lo a partir do proximo tick. * Durante a requisicao, o card recebe a classe ``opacity-50`` como feedback visual de "atualizando...". * Em caso de erro de rede, o erro e logado no ``console`` e o proximo tick tenta de novo (60s depois). Nao ha backoff agressivo para nao mascarar falhas transitorias do servidor de dev. {% endcomment %}
